LONGER PROJECT DESCRIPTION (for portfolio or LinkedIn)
----------------------------------------------------------

  Semantic Image Search Engine
  ----------------------------
  Designed and built an end-to-end semantic image search system
  that allows users to search a local image collection using plain
  English descriptions or by uploading a reference image.

  The core of the system uses OpenAI's CLIP (Contrastive
  Language-Image Pretraining) model — loaded via HuggingFace
  Transformers — to encode both images and text into a shared
  512-dimensional embedding space. Because CLIP is trained on
  400M image-text pairs, text descriptions and visually matching
  images are projected close together, enabling zero-shot
  cross-modal retrieval without any fine-tuning.

  Image embeddings are stored in a FAISS vector index (supporting
  Flat, IVF, and IVFPQ configurations) for fast approximate
  nearest-neighbor search. Embeddings are L2-normalized so that
  inner-product search is equivalent to cosine similarity.

  A Streamlit web application provides a clean browser-based UI
  where users can switch between text and image query modes,
  adjust the number of results, and view ranked results with
  similarity scores in a responsive grid layout.

  An evaluation module computes standard IR metrics (Precision@k,
  Recall@k, mAP@k, Hit Rate@k) against ground-truth labels,
  allowing systematic benchmarking of retrieval quality.

----------------------------------------------------------
KEY CONCEPTS TO MENTION IN AN INTERVIEW
----------------------------------------------------------

  - Contrastive learning / CLIP: trained to align image and text
    representations in a shared latent space
  - Vector similarity search: cosine similarity via inner product
    on L2-normalized embeddings
  - FAISS: Facebook AI Similarity Search library; Flat = exact,
    IVF = inverted file index (faster, approximate),
    IVFPQ = product quantization (memory-efficient)
  - Batch inference with PyTorch (torch.no_grad, GPU auto-detect)
  - Information retrieval metrics: P@k, R@k, mAP@k, Hit Rate@k
  - Streamlit: fast Python-native web UI framework
